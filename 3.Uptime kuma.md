# 📊 Uptime Kuma Monitoring System

## Introduction

**Uptime Kuma** is an open-source, self-hosted monitoring platform designed to monitor the availability and health of network devices and services.

It supports multiple monitoring methods, including:

* ICMP (Ping)
* HTTP / HTTPS
* TCP
* DNS
* Custom monitoring checks

In this project, Uptime Kuma serves as the **primary monitoring engine**, continuously monitoring network devices and triggering WhatsApp notifications whenever a device changes state (UP or DOWN).

---

# Architecture

Uptime Kuma is deployed using the following architecture:

```text
Host Machine
      │
      ▼
VMware Workstation
      │
      ▼
Ubuntu Server
      │
      ▼
Docker Engine
      │
      ▼
Uptime Kuma Container
```

### Benefits of this Architecture

* Service isolation from the host operating system
* Easy deployment and maintenance
* Independent container management
* Seamless integration with Node.js middleware and WAHA

# Docker Deployment

Uptime Kuma is deployed inside Docker on Ubuntu Server.

## Create Project Directory

```bash
mkdir uptime-kuma
cd uptime-kuma
```

This directory stores Docker configuration files and persistent application data.

---

## Run the Uptime Kuma Container

```bash
docker run -d \
  --restart=always \
  -p 3001:3001 \
  -v uptime-kuma:/app/data \
  --name uptime-kuma \
  louislam/uptime-kuma:latest
```

### Command Explanation

| Option                     | Description                                   |
| -------------------------- | --------------------------------------------- |
| `-d`                       | Run container in background                   |
| `--restart=always`         | Automatically restart after reboot or failure |
| `-p 3001:3001`             | Publish dashboard on port 3001                |
| `-v uptime-kuma:/app/data` | Persistent storage                            |
| `--name uptime-kuma`       | Container name                                |

# Adding a Network Monitor

To monitor a network device:

1. Click **Add New Monitor**
2. Select **Ping**
3. Enter the target IP address

Example:

```text
192.168.100.10
```

4. Set the monitoring interval (e.g., 60 seconds)
5. Click **Save**

Uptime Kuma will begin continuous ICMP monitoring of the selected device.

---

# Monitoring Behavior

Each monitoring cycle records:

* Response time
* Device status (UP / DOWN)
* Uptime percentage
* Historical latency

A device is marked **DOWN** only after failing the configured health checks.

# WhatsApp Alert Integration

Uptime Kuma sends webhook notifications to the Node.js middleware whenever a monitored device changes state.

### Notification Configuration

| Setting           | Value                                            |
| ----------------- | ------------------------------------------------ |
| Notification Type | Webhook                                          |
| Friendly Name     | WhatsApp Alerts                                  |
| Method            | POST                                             |
| URL               | `http://192.168.100.10:4000/webhook/uptime-kuma` |
| Request Body      | `application/json`                               |

### Additional Header

```json
{
  "x-api-key": "KUMA_9xA72_Secure"
}
```

The Node.js middleware receives this JSON payload and forwards formatted WhatsApp notifications through WAHA.

---

# State Change Logic

Alerts are generated **only when a device changes state**.

| Previous State | Current State | Alert |
| -------------- | ------------- | ----- |
| UP             | DOWN          | ✅ Yes |
| DOWN           | UP            | ✅ Yes |
| UP             | UP            | ❌ No  |
| DOWN           | DOWN          | ❌ No  |

This approach prevents repeated notifications and reduces alert fatigue.
