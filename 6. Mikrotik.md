# 🌐 MikroTik Router Configuration Overview

## Introduction

This section describes a **secure and structured MikroTik RouterOS configuration** designed for a small enterprise/lab environment.

It includes:

* WAN/LAN segmentation
* NAT configuration
* DNS setup
* Firewall security policies
* Port scan detection
* DDoS mitigation
* Administrative access control

The goal is to simulate a **real-world secure network perimeter**.

---

# Device Access Credentials

The router is configured with the following credentials:

* **Username:** `admin`
* **Password:** `admin123`

> ⚠️ In production environments, these credentials must be changed immediately for security reasons.

---

# Interface Configuration

Ethernet interfaces are enabled and organized into WAN and LAN roles.

```rsc id="mk_if_01"
/interface ethernet
set [ find default-name=ether1 ] disable-running-check=no
set [ find default-name=ether2 ] disable-running-check=no
```

---

## Interface Lists

```rsc id="mk_if_02"
/interface list
add name=WAN
add name=LAN
```

---

## Assign Interfaces to Lists

```rsc id="mk_if_03"
/interface list member
add interface=ether1 list=WAN
add interface=ether2 list=LAN
```

---

# Network Discovery

LAN-side discovery is enabled for easier management.

```rsc id="mk_nd_01"
/ip neighbor discovery-settings
set discover-interface-list=LAN
```

---

# IP Addressing

LAN interface is assigned a static IP:

```rsc id="mk_ip_01"
/ip address
add address=192.168.100.1/24 interface=ether2 network=192.168.100.0
```

---

# Internet Connectivity

WAN interface receives IP dynamically via DHCP:

```rsc id="mk_dhcp_01"
/ip dhcp-client
add interface=ether1 use-peer-dns=no
```

---

# DNS Configuration

External DNS servers are configured for name resolution:

```rsc id="mk_dns_01"
/ip dns
set allow-remote-requests=yes servers=8.8.8.8,1.1.1.1
```

# Firewall Configuration

## INPUT Chain (Router Protection)

```rsc id="mk_in_01"
/ip firewall filter
add action=accept chain=input connection-state=established,related comment="INPUT Allow established"
add action=drop chain=input connection-state=invalid comment="INPUT Drop invalid"

add action=accept chain=input protocol=tcp dst-port=8291 src-address=192.168.100.5 comment="Winbox Admin Only"

add action=add-src-to-address-list chain=input protocol=tcp connection-state=new \
    tcp-flags=syn address-list=attackers address-list-timeout=1d \
    comment="DDoS detection"

add action=add-src-to-address-list chain=input protocol=tcp in-interface-list=WAN psd=21,3s,3,1 \
    address-list=attackers address-list-timeout=1d comment="Port scan detection"

add action=drop chain=input comment="Drop all other traffic"
```

---

## FORWARD Chain (Traffic Control)

```rsc id="mk_fw_01"
/ip firewall filter
add action=accept chain=forward connection-state=established,related comment="Allow established"
add action=drop chain=forward connection-state=invalid comment="Drop invalid"

add action=accept chain=forward in-interface-list=LAN out-interface-list=WAN comment="LAN → WAN allowed"
add action=drop chain=forward in-interface-list=WAN out-interface-list=LAN comment="Block WAN → LAN"

add action=drop chain=forward comment="Drop all remaining traffic"
```

---

# NAT Configuration

```rsc id="mk_nat_01"
/ip firewall nat
add action=masquerade chain=srcnat out-interface=ether1
```

---

# RAW Firewall (Early Drop Layer)

RAW rules improve performance by dropping malicious traffic before connection tracking.

```rsc id="mk_raw_01"
/ip firewall raw
add action=accept chain=prerouting src-address=192.168.100.5 comment="Allow Admin First"

add action=drop chain=prerouting in-interface-list=WAN \
    src-address-list=attackers comment="Drop attackers early"
```

---

# Service Hardening

Unused services are disabled to reduce attack surface:

```rsc id="mk_srv_01"
/ip service
set ftp disabled=yes
set ssh address=192.168.100.5/32 disabled=yes
set telnet disabled=yes
set www disabled=yes
set winbox address=192.168.100.5/32
set api disabled=yes
set api-ssl disabled=yes
```

---

# Logging Configuration

Firewall logging is enabled for monitoring security events:

```rsc id="mk_log_01"
/system logging
add topics=firewall
```

---

# MAC Server Security

MAC-based access is restricted to LAN only:

```rsc id="mk_mac_01"
/tool mac-server
set allowed-interface-list=LAN

/tool mac-server mac-winbox
set allowed-interface-list=LAN
```
