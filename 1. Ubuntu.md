# 🖥️ System Implementation

## Ubuntu Server Setup and Configuration

### Introduction

Ubuntu Server is a Linux-based operating system widely used for enterprise servers due to its stability, security, flexibility, and strong community support. It is commonly deployed for hosting web services, databases, monitoring platforms, and network applications.

In this project, **Ubuntu Server** serves as the **central monitoring and alert processing server**, hosting Docker-based services, monitoring tools, and middleware responsible for real-time network monitoring and WhatsApp alert notifications.

---

## Why Ubuntu Server?

Ubuntu Server was selected because it provides:

*  Open-source and free to use
*  Stable and reliable for 24/7 operations
*  Excellent Docker support
*  Easy static IP configuration
*  Strong networking capabilities
*  Widely used in enterprise and cloud environments
*  Suitable for secure, firewall-protected deployments

# Network Configuration

## Static IP Configuration

To ensure stable communication, Ubuntu Server was configured with a static IP address.

| Parameter       | Value            |
| --------------- | ---------------- |
| IP Address      | `192.168.100.10` |
| Subnet Mask     | `/24`            |
| Default Gateway | `192.168.100.1`  |
| DNS Server 1    | `8.8.8.8`        |
| DNS Server 2    | `1.1.1.1`        |

Static IP addressing was configured using **Netplan**, Ubuntu's default network management utility.

---

## Netplan Configuration

**Configuration File**

```text
/etc/netplan/static-hostonly.yaml
```

### Configuration

```yaml
network:
  version: 2
  renderer: networkd

  ethernets:
    ens33:
      dhcp4: no

      addresses:
        - 192.168.100.10/24

      nameservers:
        addresses:
          - 8.8.8.8
          - 1.1.1.1

      routes:
        - to: default
          via: 192.168.100.1
```

---

## Purpose of Netplan Configuration

The Netplan configuration performs the following functions:

* Disables DHCP
* Assigns a permanent static IP address
* Configures the default gateway
* Defines DNS servers
* Routes all outbound traffic through the MikroTik firewall

This ensures secure and predictable network communication.

---

## Apply Netplan Configuration

Create the configuration file:

```bash
sudo nano /etc/netplan/static-hostonly.yaml
```

Apply the configuration:

```bash
sudo netplan apply
```

Verify the configuration:

```bash
ip a
ip route
```

# Ubuntu Server Access Restriction

## Objective

Administrative access is restricted so that only the authorized **Admin PC** can access Ubuntu Server.

| Device   | IP Address      |
| -------- | --------------- |
| Admin PC | `192.168.100.5` |

All other devices are denied administrative access.

---

## SSH Access Restriction

### Configuration File

```text
/etc/ssh/sshd_config
```

Add the following line:

```text
AllowUsers amir@192.168.100.5
```

Restart the SSH service:

```bash
sudo systemctl restart ssh
```

---

## Explanation

This configuration:

* Allows only the user **amir**
* Restricts access to IP address **192.168.100.5**
* Blocks SSH access from all other devices

---

# Security Outcome

After applying the configuration:

*  Only the authorized Admin PC (`192.168.100.5`) can access Ubuntu Server.
*  Unauthorized SSH access is blocked.
*  Administrative access is restricted to a single trusted system.

---

# Security Considerations

The Ubuntu Server follows several security best practices:

* Private Host-Only network deployment
* No direct Internet exposure
* All external traffic routed through MikroTik CHR
* Static IP for stable firewall and NAT rules
* Docker container isolation to reduce system-level risk
