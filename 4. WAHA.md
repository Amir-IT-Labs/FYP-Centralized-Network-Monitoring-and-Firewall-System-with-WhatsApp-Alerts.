# 💬 WAHA (WhatsApp HTTP API) Integration

## Introduction

**WAHA (WhatsApp HTTP API)** is a self-hosted platform that exposes WhatsApp functionality through secure HTTP APIs. It enables applications to send and receive WhatsApp messages programmatically without relying on third-party cloud providers.

In this project, WAHA serves as the **final message delivery layer**, forwarding alerts generated by the monitoring system directly to WhatsApp.

---

# Architecture

WAHA is deployed inside Docker on Ubuntu Server using the following architecture:

```text
Host Machine
      │
      ▼
VMware Workstation
      │
      ▼
Ubuntu Server
      │
      ▼
Docker Engine
      │
      ▼
WAHA Container
```

Unlike Uptime Kuma, WAHA **does not receive requests directly** from the monitoring system.

Instead, all requests pass through the Node.js middleware:

```text
Uptime Kuma
      │
      ▼
Node.js Middleware
      │
      ▼
WAHA API
      │
      ▼
WhatsApp
```

This design improves both security and message processing.

WAHA was selected for the following reasons:

# WAHA Installation

Before running WAHA, credentials must be initialized.

---

## Create WAHA Directory

```bash
mkdir waha
cd waha
```

This directory stores:

* Environment files
* API credentials
* Configuration files

---

## Initialize WAHA Credentials

Run the following command:

```bash
docker run --rm \
  -v "$(pwd)":/app/env \
  devlikeapro/waha init-waha /app/env
```

### Purpose

This initialization process automatically generates:

* API Key
* Administrator Username
* Administrator Password

These credentials are required for:

* Dashboard login
* API authentication
* Swagger authorization

---

# Run WAHA Container

After initialization, start the WAHA container:

```bash
docker run -d \
  --name waha \
  --restart=always \
  -p 3000:3000 \
  -v $(pwd):/app/env \
  devlikeapro/waha
```

### Command Explanation

| Option               | Description                         |
| -------------------- | ----------------------------------- |
| `-d`                 | Run in background                   |
| `--restart=always`   | Restart automatically after reboot  |
| `-p 3000:3000`       | Publish dashboard and API           |
| `-v $(pwd):/app/env` | Mount credentials and configuration |
| `--name waha`        | Container name                      |

---

# Access WAHA Dashboard

After deployment, open the dashboard:

```text
http://192.168.100.10:3000/dashboard
```

The dashboard provides:

* API status
* Session management
* Message testing
* System health

Access requires the credentials generated during initialization.

---

# WhatsApp Session Management

WAHA connects to WhatsApp using sessions.

| Setting        | Value     |
| -------------- | --------- |
| Session Name   | `default` |
| Session Status | `WORKING` |

> **Note:** The Community/Core version of WAHA supports a single active WhatsApp session.

---

# API Security

WAHA APIs are protected using an API key.

### Security Features

* API key required for every request
* Unauthorized requests are rejected
* Credentials stored securely
* API key never exposed publicly

Only the Node.js middleware is authorized to communicate with WAHA.

---

# Testing WAHA with Swagger

WAHA includes a built-in Swagger interface for testing its REST APIs.

Access Swagger:

```text
http://192.168.100.10:3000/swagger
```

### Authorization Steps

1. Open Swagger UI.
2. Click **Authorize**.
3. Enter the API key.
4. Confirm authorization.

After successful authentication, all API endpoints become available for testing.

---

# Send a Test WhatsApp Message

To verify the integration:

1. Select the **sendText** endpoint.
2. Enter the session name (`default`).
3. Enter the recipient's WhatsApp number.
4. Type a test message.
5. Click **Execute**.

A successful response confirms:

* WAHA is connected to WhatsApp.
* API authentication is working.
* Message delivery is functioning correctly.

# Integration with Node.js Middleware

WAHA never receives webhook requests directly from Uptime Kuma.

Instead, the communication flow is:

```text
Uptime Kuma
      │
      ▼
Node.js Middleware
      │
      ▼
WAHA REST API
      │
      ▼
WhatsApp User
```

### Benefits

* Additional security layer
* Message formatting before delivery
* Alert filtering
* Controlled API access

---

# Security Considerations

The following security measures were implemented:

* Docker container isolation
* Dashboard authentication
* API key authorization
* Session-based WhatsApp access
* Only port **3000** exposed
* No direct access from Uptime Kuma

---

# Role of WAHA

Within this project, WAHA functions as:

* WhatsApp message delivery engine
* REST API gateway
* Final alert delivery component
* Secure communication bridge between middleware and WhatsApp

Without WAHA, automated WhatsApp notifications would not be possible.

---

# Summary

WAHA provides a secure, lightweight, and self-hosted solution for integrating WhatsApp messaging into the monitoring platform.

Its Docker-based deployment, credential protection, API security, and Swagger testing capabilities make it suitable for production-style monitoring and alerting systems.

Combined with Node.js middleware and Uptime Kuma, WAHA enables reliable real-time WhatsApp notifications for monitored network events.
