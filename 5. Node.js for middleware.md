# ⚙️ Node.js Middleware for Alert Processing and WhatsApp Integration

## Introduction

The **Node.js middleware** is a custom-built alert processing service that connects **Uptime Kuma** with **WAHA (WhatsApp HTTP API)**.

Its main responsibilities are:

* Receiving webhook alerts from Uptime Kuma
* Authenticating incoming requests
* Formatting alerts into human-readable messages
* Forwarding messages to WAHA for WhatsApp delivery

### Key Design Goals

* Secure communication using API keys
* Separation of monitoring and messaging logic
* Controlled and formatted alert delivery

---

# System Architecture

The middleware runs on Ubuntu Server and processes webhook requests from Uptime Kuma.

```text id="mw_arch_01"
Uptime Kuma
      │
      ▼
Node.js Middleware
      │
      ▼
WAHA API
      │
      ▼
WhatsApp
```

> Uptime Kuma never communicates directly with WAHA.
> This improves security, control, and modularity.

# API Key Authentication

```javascript id="mw_auth_01"
app.use((req, res, next) => {
    const apiKey = req.headers['x-api-key'];

    if (!apiKey || apiKey !== process.env.UPTIME_KUMA_API_KEY) {
        return res.status(401).json({ error: 'Unauthorized' });
    }

    next();
});
```

## Functionality

* Validates `x-api-key` header
* Rejects unauthorized requests (HTTP 401)
* Prevents external abuse of webhook endpoint

# Message Formatting Function

```javascript id="mw_format_01"
function formatAlert(alert) {
    const monitorName = alert.monitor?.name || 'Unknown';
    const host = alert.monitor?.hostname || 'N/A';
    const statusCode = alert.heartbeat?.status;

    const statusText = statusCode === 1 ? '🟢 UP' : '🔴 DOWN';

    return `
Monitor: ${monitorName}
Host: ${host}
Status: ${statusText}
Timestamp: ${new Date().toLocaleString()}
`;
}
```

## Output Includes

* Monitor name
* Host/IP
* Status (UP / DOWN)
* Timestamp

# Sending Message to WAHA

```javascript id="mw_waha_01"
async function sendToWAHA(chatId, message) {
    await axios.post(
        `${process.env.WAHA_URL}/api/sendText`,
        {
            session: "default",
            chatId,
            text: message
        },
        {
            headers: {
                "X-API-Key": process.env.WAHA_API_KEY
            }
        }
    );
}
```

## Explanation

* Sends message via WAHA API
* Uses default WhatsApp session
* Authenticates using API key
